Admin Access Control

Restrict Admin to Localhost Only
Blocks remote access to /admin routes unless an IP allow list is configured below
IP Allow List
Enter IP addresses or CIDR subnets (one per line or comma-separated) that are allowed to access the admin panel, even if Localhost Only is enabled. Localhost (127.0.0.1, ::1) is always permitted. Leave blank for no extra IPs.
Examples: 192.168.1.50, 10.8.0.0/24, 172.16.0.0/12 Allowed IPs / Subnets One entry per line or comma-separated. Saved on blur.
Disable Admin Password Login
Hides the password form on /admin and rejects all password-based login attempts. Admin access remains available via Sign in with TLR Account (System Admin users) and Central Management one-click login.
⚠ Only enable after confirming a System Admin user exists — otherwise you will be locked out.
