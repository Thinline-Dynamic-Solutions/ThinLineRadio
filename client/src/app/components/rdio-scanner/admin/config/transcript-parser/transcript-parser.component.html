Note: This is optimized for fire department dispatches

The transcript parser identifies apparatus units and dispatch channels in call transcripts and highlights them in the UI. Configure the word lists below to match the terminology used in your area, then click Save. Changes apply retroactively; past calls are re-annotated on the next view.

How it works
  1. Corrections are applied first: common STT mis-transcriptions (e.g. SHORT HALL → SHORT FALL) are fixed before any parsing runs.
  2. Units are matched as [Prefix] Unit# Number. An optional prefix (e.g. MEDIC, BRUSH) can precede the unit type. Examples: ENGINE 5, MEDIC LADDER 3, BRUSH ENGINE 2.
  3. Channels are matched as Dispatch [Separator] Number. If channel separators are configured the format is CITY FIRE 3; without separators it is CITY 3. Shorthands let abbreviated labels like CF 3 map to a full dispatch name.
  4. All matching supports fuzzy recognition. Set Max Distance to 1 or 2 to catch common STT mishearings like ENGNE for ENGINE. Set it to 0 for exact-only matching. Aliases are alternative spellings that are treated as exact matches regardless of distance.
Unit Types {{ config.unitTypes.length }} configured

The apparatus type portion of a unit identifier, the word that comes before the unit number (and after the optional prefix).
Examples: ENGINE, TRUCK, LADDER, RESCUE, BATTALION, HAZMAT, MEDIC.
Use Aliases for alternate spellings the STT engine produces (TREK → TRUCK). Use Reject if preceded by to avoid false positives, e.g. add ELEVATOR to prevent matching RESCUE 160 in "ELEVATOR RESCUE 160 SOUTH MAIN ST".

Unit Prefixes {{ config.unitPrefixes.length }} configured

Optional words that appear before a unit type and are included in the highlighted span. Multi-word prefixes are supported.
Examples: MEDIC (as in MEDIC ENGINE 5), HEAVY, TACTICAL.

Dispatch Names {{ config.dispatchNames.length }} configured

The agency or zone name that starts a channel reference.
Examples: CITY, OPS, COUNTY, METRO.
Use aliases for common mis-transcriptions of the agency name (VEC, DECK → VECC).

Channel Separators {{ config.channelSeparators.length }} configured

The word between a dispatch name and a channel number. Leave this list empty if your channels are formatted as DISPATCH NUMBER (e.g. CITY 3). When configured the expected format is DISPATCH SEPARATOR NUMBER.
Examples: FIRE (→ CITY FIRE 3), POLICE (→ METRO POLICE 2), EMS, OPS.

Channel Shorthands {{ config.channelShorthands.length }} configured

Abbreviated channel labels that map to a full dispatch name. Shorthands are matched exactly and support space, dash, or no separator before the number (SF 3, SF-3, and SF3 all work).
Label: the short text as spoken/transcribed (e.g. SF, BACKFIRE, X-FIRE).
Dispatch: the canonical agency name to store (e.g. SANDY, VECC).
Separator (optional): the separator word to record alongside the channel (e.g. FIRE). Leave blank if not applicable.
