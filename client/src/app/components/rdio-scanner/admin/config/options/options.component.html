Hallucination Removal Patterns
Phrases that should be removed from transcripts (case-insensitive). Whisper may hallucinate common phrases from tone-only audio. Add one pattern per line. Example: "THE BELL IS INVITED TO SOUND THREE TIMES"

Automatic Hallucination Detection
What it does: Whisper sometimes "hallucinates" phrases when transcribing tone-only audio (e.g., "THE BELL IS INVITED TO SOUND THREE TIMES"). This system automatically detects these fake phrases by tracking which transcripts pass vs fail voice detection.

How it works: When a transcript fails voice detection (no speech detected in audio), its text is flagged. If the same phrase is flagged by multiple different users across multiple calls, it's likely a hallucination and will be automatically removed from future transcripts.

Learning mode: Collects hallucination data without auto-removal — review patterns in the database first, then switch to Auto-removal mode.
Auto-removal mode: Automatically removes detected hallucinations from transcripts in real-time.

Disabled Learning (collect data, don't remove) Auto-removal (remove detected hallucinations)

Confidence Threshold (0.0 - 1.0)
Minimum hallucination detection confidence to trigger auto-removal. Higher = fewer false positives but may miss some hallucinations. Lower = catches more hallucinations but may accidentally remove valid phrases.

Recommended values:
• 0.95+ (very strict) — only remove phrases with overwhelming evidence
• 0.85-0.95 (balanced) — good default for most systems
• 0.70-0.85 (aggressive) — remove more aggressively

What affects confidence:
• How many different users flagged the phrase
• How many times it was flagged
• Whisper hallucinations

Note: This only applies in Auto-removal mode. Learning mode collects data regardless of threshold.

Recommended: 0.85-0.95. Higher = fewer false positives.
